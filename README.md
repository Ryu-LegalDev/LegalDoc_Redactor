# LegalDoc Redactor

한글(HWPX) 법률 서면에서 개인정보를 자동으로 탐지하고 제거하는 로컬 AI 도구입니다.

클라우드 서비스 없이 **Ollama 로컬 LLM + 정규식 + OCR**을 조합한 하이브리드 방식으로, 민감한 법률 문서를 외부에 전송하지 않고 안전하게 처리합니다.

## 주요 기능

### 텍스트 개인정보 제거
- **정규식 기반**: 주민등록번호, 전화번호, 계좌번호, 사건번호, 법무법인명
- **LLM 기반** (Ollama gemma3:4b): 사람 이름, 장소, 상호명 등 문맥 의존적 개인정보
- **오탐 필터링**: "원고", "피고" 등 소송 역할 호칭이나 법원명을 개인정보로 오인하지 않음
- **금액 보존**: 위자료, 손해배상액 등 금액 정보는 마스킹하지 않음

### 이미지 내 개인정보 제거
- **EasyOCR** 기반 이미지 텍스트 탐지 (녹취록 캡처 등)
- 단순 검정 박스가 아닌 **당사자 역할명으로 대체** (예: 실명 → "피고")
- 복수 당사자 자동 번호 부여 (원고, 원고2, 원고3...)

### 법무법인 브랜딩 제거
- 마스터페이지 장식 요소 (파란색 테두리, 로고 등) 제거
- 직인/도장 이미지 제거
- 법무법인명 텍스트 마스킹

### 출력
- 원본과 같은 폴더에 `(파일명)_제거완.hwpx`로 저장
- HWPX 파일 구조 보존 (한글에서 정상적으로 열림)

## 처리 파이프라인

```
HWPX 파일
  ├─ 1단계: 정규식 → 주민번호, 전화번호, 사건번호 등 구조화된 패턴 탐지
  ├─ 2단계: Ollama LLM → 사람 이름, 장소, 법인명 등 문맥 의존적 탐지
  ├─ 3단계: 당사자 매핑 → 헤더에서 원고/피고/소외인 이름 자동 매핑
  ├─ 4단계: 텍스트 치환 → 본문 XML + 미리보기 텍스트 마스킹
  ├─ 5단계: 이미지 OCR → 이미지 속 이름을 역할명으로 대체
  └─ 6단계: 브랜딩 제거 → 로고, 직인, 페이지 장식 제거
       ↓
  (파일명)_제거완.hwpx
```

## 설치

### 사전 요구사항

- **Windows 10/11**
- **Python 3.10+** — https://www.python.org/downloads/ (설치 시 "Add Python to PATH" 체크)
- **Ollama** — https://ollama.com/download

### 설치 순서

```bash
# 1. Ollama 설치 후 모델 다운로드 (~2.5GB)
ollama pull gemma3:4b

# 2. 저장소 클론
git clone https://github.com/Ryu-LegalDev/LegalDoc_Redactor.git
cd LegalDoc_Redactor

# 3. Python 패키지 설치
pip install -r requirements.txt
```

> EasyOCR 첫 실행 시 한국어 OCR 모델이 자동 다운로드됩니다 (~100MB).

## 사용법

### GUI (더블클릭)

`redactor_gui.pyw`를 더블클릭하면 파일 선택 GUI가 열립니다.

1. **"파일 선택 후 실행"** 버튼 클릭
2. HWPX 파일 선택 (여러 개 동시 선택 가능)
3. 콘솔 창에서 처리 진행 상황 확인
4. 완료 알림 → 같은 폴더에 `_제거완.hwpx` 파일 생성

### CLI

```bash
# 파일 1개
python -X utf8 hwpx_redactor.py "준비서면.hwpx"

# 여러 파일
python -X utf8 hwpx_redactor.py "서면1.hwpx" "서면2.hwpx" "서면3.hwpx"
```

### 실행 예시

```
본문 길이: 7218자

[1단계: 정규식] 3건 탐지:

  [사건번호] 20**다*****
  [법인명] 법무법인 **

[2단계: LLM (gemma3:4b)] 분석 중...
  청크 1/10 처리 중...
  ...

[2단계: LLM] 6건 탐지:
  [사람이름] O O O
  [사람이름] O O O
  [사람이름] OOO
  [장소] **시
  [장소] **나이트클럽
  [사람이름] OO

[당사자 매핑]
  OOO → 원고
  OOO → 피고
  OOO → 소외인
  OO → 관계인

[3단계: 파일 처리]
  이미지 OCR 마스킹: BinData/image1.png
  이미지 OCR 마스킹: BinData/image2.png
  로고 제거: BinData/image3.png
  마스터페이지 정리: Contents/masterpage0.xml
  직인 제거: BinData/image4.png
```

## 탐지 대상

| 유형 | 탐지 방식 | 예시 |
|------|-----------|------|
| 주민등록번호 | 정규식 | `880101-1234567` |
| 전화번호 | 정규식 | `010-1234-5678` |
| 계좌번호 | 정규식 | `110-123-456789` |
| 사건번호 | 정규식 | `2024가소123456` |
| 법무법인명 | 정규식 | `법무법인 OO` |
| 사람 이름 | LLM | `홍길동`, `홍 길 동` |
| 장소/상호 | LLM | `OO시`, `OO나이트클럽` |
| 이미지 속 이름 | OCR + LLM 매핑 | 녹취록 캡처 내 이름 → 역할명 대체 |

## HWPX 파일 구조

HWPX는 한글 워드프로세서의 문서 형식으로, ZIP 아카이브 안에 XML 파일들로 구성됩니다.

```
파일.hwpx (ZIP)
  ├─ Contents/
  │    ├─ section0.xml    ← 본문 텍스트 (<hp:t> 태그)
  │    ├─ masterpage0.xml ← 페이지 장식, 로고
  │    └─ header.xml      ← 스타일, 배경 이미지 참조
  ├─ BinData/
  │    ├─ image1.png      ← 녹취록 캡처 등 삽입 이미지
  │    ├─ image3.png      ← 법무법인 로고
  │    └─ image4.png      ← 직인/도장
  └─ Preview/
       └─ PrvText.txt     ← 미리보기용 평문 텍스트
```

## 주의사항

- **Ollama가 백그라운드에서 실행 중**이어야 합니다 (시스템 트레이 아이콘 확인)
- 처리할 파일이 **한글(Hwp)에서 열려있으면** PermissionError가 발생합니다
- LLM은 비결정적이므로 **실행할 때마다 결과가 약간 다를 수 있습니다**
- PowerShell에서 한글 출력 깨짐 방지를 위해 `-X utf8` 플래그를 사용합니다

## 기술 스택

- **Python 3.10+**
- **Ollama** + **gemma3:4b** — 로컬 LLM (한국어 NER)
- **EasyOCR** — 이미지 내 한국어 텍스트 좌표 탐지
- **Pillow** — 이미지 마스킹 및 텍스트 렌더링
- **lxml** — HWPX XML 파싱 및 수정
